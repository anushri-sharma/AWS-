# [AWS Well-Architected Framework](https://skillbuilder.aws/renderer/?module_id=B4R4AU7BWJ%3A001.000.000&product_id=8D79F3AVR7%3A002.002.003&registration_id=3a5edca9-7621-5cd1-8fb0-7734f046051b&referrer=https%3A%2F%2Fskillbuilder.aws%2Flearn%2F94T2BEN85A%2Faws-cloud-practitioner-essentials%2F8D79F3AVR7&navigation=digital)

### A development team is releasing frequent updates to an application hosted on AWS. Right now, each deployment requires manual steps that often result in downtime or inconsistent environments.

### What AWS practice from the Well-Architected Framework could help in this situation?
> **Automate deployments through a continuous integration and continuous delivery pipeline (Operational Excellence).**
> Automating deployments with a CI/CD pipeline like AWS CodePipeline would streamline the process, reduce downtime, and help provide more reliable deployments.
----

### What is the purpose of the AWS Well-Architected Tool (AWS WA Tool)?
> **To assess and improve cloud workloads based on six key architectural pillars**
> The AWS WA Tool helps evaluate workloads using the six pillars: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, and Sustainability.

### A company is using a large Amazon EC2 instance type for a low-traffic website, but monitoring shows CPU usage rarely exceeds 10 percent.

### What is a good next step?

> **Rightsize the EC2 instance to a smaller type to match the actual workload.**
> Rightsizing the EC2 instance to a smaller type that fits the actual workload would improve performance efficiency by avoiding resource wastage.

Question
03/06
### A startup is hosting a critical web application on a single Amazon EC2 instance in one Availability Zone. After a brief outage, they decide to reevaluate their architecture.

### What practice from the AWS Well-Architected Framework could improve this situation?

>  **Deploy instances across multiple Availability Zones for fault tolerance (Reliability).**
> By deploying instances across multiple Availability Zones, the web application will be more resilient to failures and will have higher availability.
